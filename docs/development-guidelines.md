# Development Guidelines

## 1. 方針
- PoCとして最短で価値検証ができる実装を優先する
- ルールベースで安全に動く設計を維持する
- 医療的な表現は避け、注意書きを徹底する

## 2. コーディング規約（Dart/Flutter）
- ファイル/ディレクトリ名は `lower_snake_case`
- 型/クラスは `UpperCamelCase`
- 変数/関数は `lowerCamelCase`
- 1ファイルの責務は小さく保つ
- UIはWidget分割で再利用性を確保する

## 3. 命名規則（ドメイン）
- 用語は `docs/glossary.md` に準拠
- 「肌荒れ」「症状」「問診」「今夜の指示」などは一貫した日本語表記で扱う

## 4. スタイリング規約
- 余白/文字サイズは一定のスケールで統一
- 主要ボタンの見た目は全画面で統一
- 強い警告/注意は色と文言の両方で伝える

## 5. テスト規約（PoC）
- 重要な判定ロジックはユニットテストを優先
- UIは手動テストで十分（PoC範囲）
- 受診誘導の表示は必ず確認する

## 6. Git規約
- 1コミット=1意図
- PRには簡潔な目的/変更点を書く
- 大きな変更は先に `.steering/` を作成する

## 7. セキュリティ/プライバシー
- 個人情報は取得しない
- 端末内保存のみとする
- デバッグログに個人情報が出ないようにする
