# Requirements (0002 Define PoC Personas)

## 1. 目的
PoC（肌荒れ時に家にあるスキンケアから「今夜何を使うか」を安全に指示するiOS/Androidアプリ）の検証対象となるペルソナを策定する。メイン3人＋ネガティブ1人を定義し、各ペルソナの利用シナリオとPoCで観測するKPIを明文化する。

## 2. 出力要件
- PoCの検証仮説（価値検証）がペルソナごとに読み取れる
- 入力負担（商品名 + カテゴリ、短い問診）に耐えるユーザー像である
- 医療的リスクを避けるための対象外（ネガティブペルソナ）が含まれる
- 2週間のPoCで観測できる行動（再訪、実行、フォロー回答）を想定できる

## 3. ペルソナ一覧（合計4人）
- P1（メイン）: 季節/花粉/乾燥で荒れる敏感寄り
- P2（メイン）: 攻め成分の重ねすぎで荒れやすい
- P3（メイン）: 忙しくて時間がない、肌が不安定で困る
- N1（ネガティブ）: 医療行為や診断を求めがち/重症の可能性

## 4. ペルソナ詳細
### P1（メイン）季節/花粉/乾燥で荒れる敏感寄り
- 基本属性: 20代後半 / デスクワーク / スキンケア経験は中程度
- 肌状態: 普段は安定だが季節変わり目に荒れやすい / 乾燥と赤み / 月1〜2回
- 荒れるトリガー: 乾燥、花粉、睡眠不足
- 家にあるスキンケア量: 中（化粧水1、乳液1、保湿クリーム1、低刺激美容液1、ワセリン1）
- 困りごと（1文）: 荒れた夜に何を使うか迷って時間を無駄にする。
- このアプリに期待する「今夜の答え」（1文）: 最低限の安全ケアを短く示してほしい。
- 入力許容度: 問診は2問までOK / 手持ち登録は5個までOK
- 成功体験（このアプリが役に立ったと感じる瞬間）: 迷わず「使う/休む」が決まり、安心して眠れる。
- 失敗条件（使わなくなる理由）: 指示が長い、曖昧、または刺激に見える。
- 懸念/リスク（安全・医療誤解の観点）: 受診が必要な状態を自己判断で見逃す。
- PoCで観測する主要指標: 問診完了率、提案実行の自己申告、翌日フォロー回答率、7日以内再訪率

### P2（メイン）攻め成分の重ねすぎで荒れやすい
- 基本属性: 20代後半 / SNSで美容情報収集 / スキンケア経験は高め
- 肌状態: 普段は皮脂が出やすい / 荒れる時はヒリつきと赤み / 月2〜3回
- 荒れるトリガー: 角質ケアの重ね使い、濃い美容液の連用、睡眠不足
- 家にあるスキンケア量: 多（化粧水2、美容液3、角質ケア系2、クリーム1、日焼け止め1）
- 困りごと（1文）: 攻めすぎた夜に「何を休むか」が決められない。
- このアプリに期待する「今夜の答え」（1文）: 休むカテゴリを明確にして安全側に倒してほしい。
- 入力許容度: 問診は3問までOK / 手持ち登録は12個までOK
- 成功体験（このアプリが役に立ったと感じる瞬間）: 休止が明確で、明日悪化しなかった。
- 失敗条件（使わなくなる理由）: 休止対象が不明確、または理由が納得できない。
- 懸念/リスク（安全・医療誤解の観点）: 攻め成分の中止を医療指示と誤解する。
- PoCで観測する主要指標: 休止カテゴリへの納得度、提案実行の自己申告、翌日フォロー回答率、2回目の利用率

### P3（メイン）忙しくて時間がない、肌が不安定で困る
- 基本属性: 30代前半 / 子育て中 / スキンケア経験は中程度
- 肌状態: 普段は混合肌 / 荒れる時は乾燥とかゆみ / 月1回程度
- 荒れるトリガー: 睡眠不足、季節の乾燥、ストレス
- 家にあるスキンケア量: 少〜中（化粧水1、オールインワン1、保湿クリーム1）
- 困りごと（1文）: 時間がなく、最短で安全なケアを決めたい。
- このアプリに期待する「今夜の答え」（1文）: 2〜3ステップで終わる手順を示してほしい。
- 入力許容度: 問診は2問までOK / 手持ち登録は3個までOK
- 成功体験（このアプリが役に立ったと感じる瞬間）: 1分で手順が決まり、すぐ実行できた。
- 失敗条件（使わなくなる理由）: 入力が面倒、または手順が多すぎる。
- 懸念/リスク（安全・医療誤解の観点）: 忙しさから注意事項を読み飛ばす。
- PoCで観測する主要指標: 問診完了率、結果表示までの所要時間、提案実行の自己申告、翌日フォロー回答率

### N1（ネガティブ）医療行為や診断を求めがち/重症の可能性
- 基本属性: 年代不問 / 症状への不安が強い / スキンケア経験は不問
- 肌状態: 普段から不安定 / 荒れる時は腫れ・痛み・広範囲 / 頻度は不定
- 荒れるトリガー: 不明、急激な悪化、強い刺激への反応
- 家にあるスキンケア量: 不問（少〜多まで幅広い）
- 困りごと（1文）: 医療判断に近い答えを求めてしまう。
- このアプリに期待する「今夜の答え」（1文）: 症状の原因特定や治療指示を期待する。
- 入力許容度: 問診は1〜2問までOK / 手持ち登録は0〜2個までOK
- 成功体験（このアプリが役に立ったと感じる瞬間）: 受診誘導で早めに相談できた。
- 失敗条件（使わなくなる理由）: 医療的な答えが出ないことに失望する。
- 懸念/リスク（安全・医療誤解の観点）: 受診が必要な状態の取りこぼし。
- PoCで観測する主要指標: 受診誘導の表示率、注意事項の確認率、途中離脱率、翌日フォロー回答率

## 5. ペルソナ別の利用シナリオ（各1本）
### P1
乾燥と赤み → Homeから問診開始 → 症状は「赤み」、強度は「中」 → 手持ちは保湿中心で5個登録 → 結果画面の「休む/使う」が短く明確 → 翌日フォローで「少し落ち着いた」を選択

### P2
ヒリつきが出た → Homeから問診開始 → 症状は「ヒリつき」、強度は「強め」 → 手持ちは多めで攻め系も登録 → 結果画面の「休止カテゴリ」が刺さる → 翌日フォローで「悪化なし」を選択

### P3
夜にかゆみ → Homeから問診開始 → 症状は「かゆみ」、強度は「軽め」 → 手持ちは3個のみ登録 → 結果画面の「2ステップで終了」が刺さる → 翌日フォローで「変わらない」を選択

### N1
腫れと痛み → Homeから問診開始 → 症状は「腫れ」、強度は「強め」 → 手持ち入力は省略 → 結果画面で受診誘導が強調される → 翌日フォローは未回答（離脱）

## 6. スコープ外の明文化（N1に紐づけて）
- 対象外の症状/状態: 腫れ、水ぶくれ、強い痛み、急激な悪化、広範囲の赤み、発熱を伴う状態
- 対象外の場合にアプリが必ず行うこと: 受診誘導の表示、一般的注意の提示、攻め成分の中止を案内、入力を最小化して離脱を許容

## 7. 制約事項 / 非対象
- 医療診断や処方はしない
- 具体的な薬剤・治療指示は避け、受診誘導を含める
- 成分解析、商品DB、画像解析、外部連携には依存しない
